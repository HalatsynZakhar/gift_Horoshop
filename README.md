# Подарунки Хорошоп

Веб-сервіс призначає подарунки товарам через API Хорошоп. У полях форми та Excel вказуються `article_for_display` — артикули, які відображаються на сайті. Сервіс знаходить відповідні внутрішні `article` в експорті каталогу та передає їх Хорошопу.

## Запуск

1. У `config.json` вкажіть домен магазину та порт. Новий сервіс за замовчуванням використовує порт `8094`, щоб не конфліктувати з сервісом наборів.
2. Встановіть залежності: `pip install -r requirements.txt`.
3. Запустіть `python gifts_server.py`.
4. Відкрийте `http://<сервер>:8094`.

Логін і пароль API вводяться у веб-панелі та не зберігаються. Замість них можна використати чинний API-токен.

## Як працюють подарунки

API Хорошоп перезаписує весь масив `gifts` основного товару. Тому перед кожною операцією сервіс експортує каталог, зберігає вже призначені подарунки та:

- при додаванні дописує лише новий подарунок;
- при видаленні прибирає лише обраний подарунок;
- показує поточні зв'язки безпосередньо з експорту Хорошоп.

Це означає, що подарунки, призначені раніше вручну або іншим сервісом, не стираються.

## Excel

Шаблон містить дві колонки:

| Колонка | Значення |
| --- | --- |
| Артикул основного товару | `article_for_display` товару, до якого додається подарунок. |
| Артикул подарунка | `article_for_display` товару-подарунка. |

Вивантаження поточних подарунків містить додаткову колонку `Видалити (Так)`. Впишіть `Так` навпроти зв'язків, які треба видалити, і завантажте файл назад. Порожня клітинка означає додавання або збереження зв'язку.

Пошук артикулів виконується спочатку точним збігом `article_for_display`, потім без урахування регістру. Якщо артикул не знайдений або неунікальний, операція для цього рядка не виконується.

## Публічний лог

У `config.json` вкажіть папку, яка доступна для перегляду, та спільний файл:

```json
"logging": {
  "public_log_path": "C:/ShareFiles/public",
  "public_log_name": "horoshop_gifts.log"
}
```

У цей файл записуються повідомлення контролера оновлень, Uvicorn, API та помилки. У JSON використовуйте прямі слеші `/` або подвійні зворотні `\\`.

## Windows Server

Запускайте PowerShell від імені адміністратора:

```powershell
.\FULL_SERVER_INSTALL.ps1
```

Інсталятор створює `config.json`, зупиняє попередню версію, виконує `git reset --hard` та запускає контролер автооновлення. Контролер перевіряє Git кожні п'ять хвилин, не створює середовище заново та повертає попередню версію, якщо оновлення не пройшло перевірку.

Для повної зупинки:

```powershell
.\scripts\stop_tasks.ps1
```
